Versions for the Matrix specification follow a three-part format like major.minor.patch.

* `major` version increments are reserved for very significant changes. * `minor` version increments may signal additions, deprecations, or breaking changes. * `patch` version increments are for clarifications to the specification and have no functional effect on implementations.
